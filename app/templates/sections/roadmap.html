Траектория Python Backend

Бесплатный старт и 214 практических уроков: от Python и Git до production-доставки и защиты портфельного сервиса.

  • На бесплатной консультации определяем текущий уровень, карьерную цель и узкие места в базе. После встречи ты получаешь персональный roadmap, стартовые задачи и тестовый доступ к платформе, чтобы оценить формат обучения до начала работы.

    Результат: у тебя есть понятная точка старта, приоритеты на ближайшие недели и первый практический шаг.

  • Объединяем фундамент Python и первую проектную практику: execution model, терминал, Git, типы данных, ветвления, циклы, коллекции, функции, обработку ошибок и отладку. Затем переходим к модулям, пакетам, файловому хранению, JSON, объектной модели, dataclass, тестам и разделению ответственности.

    На протяжении этапа развиваешь Planner: от консольного прототипа до модульного приложения с сохраняемыми данными, тестами, README и GitHub Release.

    Результат: ты уверенно пишешь и отлаживаешь Python-код, ведёшь проект через Git и собираешь поддерживаемое приложение с проверяемым поведением.

  • Разбираем жизненный цикл HTTP-запроса: адресацию, методы, headers, body, status codes, Content-Type и семантику REST. Через Postman и FastAPI строим Planner API: path и query параметры, Pydantic-схемы, валидацию 422, CRUD, PATCH и DELETE, APIRouter, Swagger/OpenAPI и интеграционные тесты через TestClient.

    Результат: ты проектируешь предсказуемый HTTP API, разделяешь входные и выходные модели и доказываешь корректность контрактов автотестами.

  • Собираем FastAPI как цельное приложение: dependency injection, configuration, environment variables, middleware, CORS и обработку ошибок. Затем подключаем SQLite и SQLAlchemy: engine, session lifecycle, ORM-модели, CRUD, связи, пагинацию, транзакции и Alembic. Параллельно изучаем SQL, нормализацию, JOIN, агрегаты, PostgreSQL roles, индексы, EXPLAIN ANALYZE, backup/restore и осознанный выбор Redis или MongoDB под отдельные сценарии.

    Результат: ты проектируешь data layer, контролируешь транзакционные границы, читаешь планы запросов и переносишь API на PostgreSQL без потери контракта.

  • Выстраиваем модель идентичности и границы доступа: пользователь, пароль и безопасное хеширование, HttpOnly-cookie, server-side sessions, access/refresh tokens, ротация и отзыв токенов. Реализуем OAuth2 password flow, current_user dependency, ownership-проверки, RBAC, API keys и различие между 401 и 403. Отдельно формируем threat model и тестируем критические сценарии входа.

    Результат: ты создаёшь персонализированный API, где права, сессии и токены имеют явные границы и покрыты проверками.

  • Разбираем coroutine, event loop, конкурентные задачи, timeout, cancellation, внешние HTTP-клиенты, AsyncSession и диагностику блокирующих вызовов. Затем переносим сервис в production-подобный контур: Linux-процессы, порты, сигналы, structured logs, healthcheck, Dockerfile, Compose, PostgreSQL и Redis services, миграции, GitHub Actions, quality gates, deployment, secrets и rollback.

    Результат: ты понимаешь поведение сервиса под I/O-нагрузкой и собираешь воспроизводимый delivery pipeline от локального стека до управляемого релиза.

  • Финализируем StudyHub LMS: моделируем учебный контур, зачисление и прогресс, добавляем Redis как cache и source of truth там, где это оправдано, разбираем TTL и фоновые операции. Затем проводим финальный quality audit, оформляем README, демонстрационный сценарий и GitHub Release. Отдельно тренируем техническое интервью: Python, HTTP, SQL, FastAPI, архитектура, trade-offs и защита решений.

    Результат: у тебя есть развёртываемый capstone-проект, аргументированная история решений и материал для портфолио и технических интервью.
