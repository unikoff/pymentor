Инженерный путь в Python Backend

От диагностики и фундаментального Python до API, данных, безопасности, production-практик и собеседований.

  • Проводим технический аудит: проверяем Python, алгоритмическое мышление, Git, HTTP, базы данных и умение объяснять собственный код. Формулируем целевую позицию, ограничения по времени и измеримые контрольные точки. На основе результатов строим индивидуальную последовательность тем без дублирования уже освоенного материала.

    Результат: персональная карта компетенций, приоритеты обучения и понятные критерии перехода между этапами.

  • Последовательно разбираем модель выполнения Python, коллекции, функции и область видимости, исключения, модули и пакеты, файловое хранение, сериализацию, объектную модель, композицию и наследование. Учимся определять контракт функции, разделять ответственности, управлять зависимостями, применять type hints и фиксировать поведение автоматическими тестами на pytest.

    Результат: вы декомпозируете задачи, пишете поддерживаемый Python-код, тестируете ключевые сценарии и аргументированно объясняете принятые решения.

  • Разбираем семантику HTTP-методов и статус-кодов, идемпотентность, структуру request и response, валидацию и сериализацию через Pydantic. Проектируем CRUD-контракты, разделяем входные и выходные схемы, организуем маршруты через APIRouter, применяем dependency injection, middleware, CORS, exception handlers, OpenAPI и интеграционные тесты через TestClient.

    Результат: рабочий версионируемый API с прозрачным контрактом, документацией, единым форматом ошибок и автоматическими тестами.

  • Начинаем с SQLite и синхронного SQLAlchemy: engine, lifecycle Session, ORM-модели, identity map, unit of work, транзакционные границы и rollback. Реализуем выборки, фильтрацию, сортировку, пагинацию, связи one-to-many, eager loading и диагностику N+1. Управляем схемой через Alembic, затем изучаем SQL, нормализацию, JOIN, агрегаты, индексы, EXPLAIN и перенос проекта на PostgreSQL. Redis и MongoDB рассматриваем как отдельные модели хранения под конкретные сценарии, а не как взаимозаменяемые базы.

    Результат: вы проектируете схему данных, контролируете транзакции, читаете ORM-запросы и можете обосновать выбор технологии хранения.

  • Разделяем идентификацию, аутентификацию и авторизацию. Реализуем безопасное хеширование паролей, server-side sessions в HttpOnly-cookie, access и refresh tokens, ротацию и отзыв токенов, OAuth2 password flow в FastAPI, dependency current_user, ownership-проверки и ролевую модель доступа. Отдельно разбираем API keys, HTTP Basic, различие между ответами 401 и 403 и базовую threat model приложения.

    Результат: персонализированный API, в котором пользователь видит только свои данные, а механизмы входа и проверки прав покрыты тестами.

  • Разбираем event loop, coroutine, task, cancellation, timeout и различие между I/O-bound и CPU-bound нагрузкой. Учимся находить блокирующие вызовы, корректно использовать async/await, AsyncSession и конкурентное выполнение без потери контроля над ошибками. На уровне архитектуры развиваем модульный монолит: выделяем bounded responsibilities, сервисные границы, конфигурацию, логирование и наблюдаемость без преждевременного перехода к микросервисам.

    Результат: вы понимаете, когда асинхронность действительно повышает пропускную способность, и умеете развивать приложение без хаотичного роста связей.

  • Собираем воспроизводимое окружение через Dockerfile и Docker Compose, разделяем build-time и runtime configuration, подключаем health checks, миграции, structured logging и управление секретами. Настраиваем CI-пайплайн: линтер, тесты, сборка образа и контролируемый deploy. Затем проводим code review, разбираем типовые вопросы по Python, HTTP, SQL, FastAPI и архитектуре, решаем практические задачи, оформляем README и готовим техническую защиту проекта.

    Результат: развёртываемый портфельный сервис, прозрачная история разработки и готовность аргументированно проходить техническое интервью.
